14      /**
15       * http://www.slightlymagic.net/forum/viewtopic.php?f=70&t=16732&p=172937&hilit=Sharuum+the+Hegemon#p172920
16       * <p>
17       * My Sharuum EDH deck uses the standard Sharuum + Clone Effect + Blood Artist as one of the win
18       * conditions, but when I have Sharuum in plan and play a Clever Impersonator, targetting Sharuum
19       * and choose to keep the Clever Impersonator and send the original Sharuum to the graveyard Xmage
20       * never gives me the option to use the Sharuum Ability that the Clever Impersonator should get,
21       * making the combo not work.
22       * <p>
23       * I run a Sharuum EDH deck that wins by cloning Sharuum for infinite death triggers. I know the rules
24       * check out on this combo irl, but no matter how I stack the triggers for cloning Sharuum and her enter
25       * the battlefield effect it does not work. It either ends with Sharuum in my graveyard or the reanimate
26       * effect hits the stack before the legend rule applies
27       * <p>
28       * [1] Sharuum the Hegemon is on the battlefield.
29       * [2] You cast Clone (or any other Clone-like card).
30       * [3] When Clone resolves, you choose Sharuum for the replacement effect.
31       * [4] Since fake-Sharuum entered the battlefield, its EtB ability triggers.
32       * [5] State-based actions are checked and you are prompted to keep one Sharuum. You sacrifice real-Sharuum.
33       * 116.2a Triggered abilities can trigger at any time, including while a spell is being cast, an ability is being activated, or a spell or
34       * ability is resolving. (See rule 603, "Handling Triggered Abilities.") However, nothing actually happens at the time an ability triggers.
35       * Each time a player would receive priority, each ability that has triggered but hasn't yet been put on the stack is put on the stack. See rule 116.5
36       * 116.5. Each time a player would get priority, the game first performs all applicable state-based actions as a single event (see rule 704,
37       * "State-Based Actions"), then repeats this process until no state-based actions are performed. Then triggered abilities are put on the stack
38       * (see rule 603, "Handling Triggered Abilities"). These steps repeat in order until no further state-based actions are performed and no abilities
39       * trigger. Then the player who would have received priority does so.
40       * [6] Once State-based actions are finished, triggered abilities go on the stack. You put the EtB from [4] choosing real-Sharuum.
41       * [7] Real-Sharuum enters the battlefield.
42       * [8] Rinse and repeat.
43       */
44      @Test
45      public void testCloneTriggered() {
46          // When Sharuum the Hegemon enters the battlefield, you may return target artifact card from your graveyard to the battlefield.
47          addCard(Zone.BATTLEFIELD, playerA, "Sharuum the Hegemon", 1);
48          addCard(Zone.HAND, playerA, "Clone", 1);
49          addCard(Zone.BATTLEFIELD, playerA, "Island", 4);
50  
51          // Whenever Blood Artist or another creature dies, target player loses 1 life and you gain 1 life.
52          addCard(Zone.BATTLEFIELD, playerA, "Blood Artist", 1);
53  
54          castSpell(1, PhaseStep.PRECOMBAT_MAIN, playerA, "Clone");
55          setChoice(playerA, true);
56          setChoice(playerA, "Sharuum the Hegemon"); // what creature to clone

58          setChoice(playerA, "Sharuum the Hegemon[only copy]"); // which legend to keep
59          setChoice(playerA, "Whenever {this} or another creature dies"); // blood first
60          addTarget(playerA, playerB); // damage by blood
61          setChoice(playerA, true); // return
62          // addTarget(playerA, "Sharuum the Hegemon"); // return real sharuum (Autochosen, only target)

64          setChoice(playerA, "Sharuum the Hegemon[only copy]"); // which legend to keep
65          setChoice(playerA, "Whenever {this} or another creature dies"); // blood first

67          setChoice(playerA, true); // return
68          // addTarget(playerA, "Sharuum the Hegemon"); // return real sharuum (Autochosen, only target
69  
70          setChoice(playerA, "Sharuum the Hegemon[only copy]"); // which legend to keep
71          setChoice(playerA, "Whenever {this} or another creature dies"); // blood first
72          addTarget(playerA, playerB); // damage by blood

74          // addTarget(playerA, "Sharuum the Hegemon"); // return real sharuum (Autochosen, only target)

78          addTarget(playerA, playerB); // damage by blood
